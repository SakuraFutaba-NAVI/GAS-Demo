# GAS-Demo
UE5.7 GAS框架求职demo
基于 Unreal Engine 纯 C++ 扩展 GameplayAbilitySystem 实现的第三人称战斗 Demo，完整覆盖能力系统、属性系统、连击功能、敌人 AI、道具系统、UI 交互等。

##技术栈

引擎：Unreal Engine 5.7
开发语言：C++ 17
核心系统：GameplayAbilitySystem (GAS)、增强输入系统（Enhanced Input）
UI 系统：Unreal Motion Graphics (UMG)
网络：UE 原生网络同步、服务端权威模型
AI：行为树、AI 控制器

架构设计
项目采用「核心框架层 + 业务逻辑层 + 表现层」三层解耦结构：
核心框架层：C++ 封装 GAS 基类（ASC、AttributeSet、GA 基类）、增强输入管理、通用工具函数，与具体业务逻辑完全解耦
业务逻辑层：基于基类扩展玩家技能、连击逻辑、敌人 AI、道具系统、网络同步规则，模块独立可快速扩展
表现层：UMG 界面、动画、特效，仅负责表现效果，不写入核心战斗逻辑
核心类依赖关系：
自定义 UAbilitySystemComponent 派生基类，统一管理能力激活、标签生命周期
自定义 UAttributeSet 派生属性集，管控血量、法力、攻击力等基础数值与变更回调
自定义 AGameplayAbility 派生技能基类，封装激活 / 取消 / 打断通用流程，支持标签权限校验
核心实现细节
1. 纯 C++ 实现 GAS 三大核心体系
背景：蓝图实现 GAS 存在逻辑分散、复用性差、不利于网络同步的问题
方案：C++ 完整实现能力系统、属性系统、标签系统的底层逻辑
能力系统：封装 GA 生命周期，统一处理激活条件、消耗判定、打断规则
属性系统：基于 AttributeSet 扩展基础属性，封装数值变更回调与阈值校验
标签系统：通过 GameplayTag 管控能力权限、状态判定、逻辑分支，替代硬编码判断
效果：核心战斗逻辑高度复用，维护成本降低，天然适配 UE 网络同步模型
2. 基于双标签窗口的连击机制
背景：普攻连击需要精准控制可接续时段，超时需重置连击段数
方案：引入两个 GameplayTag 作为时间窗口标记
「窗口开启期」标签：攻击动作执行到指定帧时激活，此时按下攻击键可触发下一段连击
「窗口关闭期」标签：动作播放到尾部时激活，此时输入无效，连击计数清零
超出窗口时间未输入，自动清除连击计数，下次攻击回归第一段
效果：连击节奏精准可控，逻辑完全基于标签驱动，无需额外状态机，扩展新连击段仅需新增 GA
3. 双类型敌人 AI 与弹道设计
背景：近战与远程敌人攻击逻辑差异大，需拆分实现避免耦合
方案：
近战敌人：逻辑内聚在敌人角色类中，通过行为树驱动「寻敌→追击→攻击」循环，近身触发伤害判定
远程敌人：独立封装子弹 Actor 基类，派生两种弹道实现
抛物线弹道：模拟重力加速度，适用于弓箭类敌人
直线弹道：匀速直线飞行，适用于枪械类敌人
子弹碰撞后触发伤害事件，联动 GAS 属性系统扣减生命值
效果：两类敌人逻辑完全解耦，可快速新增敌人类型与弹道样式
4. 道具与属性联动机制
背景：回复道具需要同时修改多个属性，且需适配联机同步
方案：基于 GAS GameplayEffect 实现道具效果
回复药道具使用时，触发对应 GE，同时修改生命值、法力值 Attribute
道具逻辑与属性系统解耦，新增道具仅需配置对应 GE 参数
效果：属性变更统一走 GAS 数值管线，天然支持服务端权威同步，无状态不一致问题
5. 击杀重生与全状态重置
背景：敌人重生后易出现属性残留、Buff 标签未清除、AI 状态异常
方案：死亡触发时批量移除所有动态 GameplayTag 与临时 GE；重生节点调用 AttributeSet 初始化接口还原基础数值，同时重置 AI 行为树与运动状态
效果：重生后敌人状态完全初始化，无数据残留与逻辑异常
